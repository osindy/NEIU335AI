# Group Project Submission Guide

## Overview
This repository uses a **Pull Request (PR)** workflow for group project submissions. This allows for open collaboration, where teams can see and learn from each other's work.

## Submission Instructions

### 1. Fork the Repository
- Click the "Fork" button at the top right of this page to create your own copy of the repository.

### 2. Create a Branch
- In your forked repository, create a new branch for your team's work.
- **Branch Naming Convention:** `group-project-TeamName` (e.g., `group-project-AlphaAI`)

### 3. Create Your Project Folder
- Navigate to the `Group_Projects` directory.
- copy the `Template` folder and rename it to your Team Name (e.g., `Team_Alpha`).
- **Do not modify other teams' folders.**

### 4. Add Your Materials
Your project folder should contain:
- **Source Code:** All Python scripts, notebooks, and assets.
- **README.md:** Filled out according to the template.
- **Presentation:** PDF or link to your slides.
- **requirements.txt:** List of library dependencies.

### 5. Submit a Pull Request (PR)
- When you are ready to submit, go to the "Pull Requests" tab in the original repository.
- Click "New Pull Request".
- Select your fork and branch as the source.
- Add a clear title and description.
- Click "Create Pull Request".

## Review Process
- Your instructor will review the code directly in the PR.
- Feedback may be given as comments on specific lines of code.
- Once approved, your project will be merged into the main branch!
